API ENDPOINTS
-------------
POST   /api/auth/register            - User registration
POST   /api/auth/login               - User login
POST   /api/upload                   - Upload & process medical report
GET    /api/reports/:id              - Get report by ID
GET    /api/vault/:patientId         - Get patient vault
POST   /api/chat                     - Chat with medical data

ENVIRONMENT SETUP
-----------------
Backend/.env:
  PORT=5000
  NODE_ENV=development
  MONGODB_URL=mongodb://localhost:27017/smart_medical_analyser
  JWT_SECRET=your_jwt_secret
  GOOGLE_GEMINI_API_KEY=your_gemini_api_key
  GEMINI_API_KEY=your_gemini_api_key

USAGE
-----
1. Start Backend Server:
   cd Backend
   npm install
   npm run dev

2. CLI Workflow:
   python main.py "data/pdfs/report.pdf"

3. Individual Agents:
   python ExtractionAgent/ExtractionAgent.py "report.pdf"
   python InsightAgent/Summary.py "report.json"
   python InsightAgent/Recommendation.py "report.json"
   python ValutAgent/ValutAgent.py "report.pdf"

WORKFLOW
--------
1. Upload PDF → /api/upload
2. ExtractionAgent → Extract data to JSON
3. InsightAgent → Generate summary + recommendations (parallel)
4. VaultAgent → Organize by patient
5. Store in MongoDB
6. ChatBot → Interactive Q&A with report

TECH STACK
----------
Backend:     Node.js, Express.js, MongoDB, Mongoose
API Auth:    JWT (JSON Web Tokens)
ML Agents:   Python 3.10+, Google Gemini AI
PDF Processing: PyMuPDF, pdfplumber, pandas
AI Models:   Gemini 2.5 Flash
